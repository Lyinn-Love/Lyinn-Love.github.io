AI 辅助 FPGA 开发：我手搓了一个 Vivado MCP Server

2026-04-05

背景与缘起

今天心血来潮，想要在 Trae IDE 里用大模型（MCP）直接控制 Vivado 进行 FPGA 开发。原本试着使用开源社区或第三方的 mcp_synthpilot，但发现其核心是一个编译好的闭源二进制黑盒，并且存在 License 限制，无法直接修改和扩展功能。

于是，在我的专属 AI 助理（一只贴心的猫娘 🐱）的陪伴下，我们决定从零开始，自己用 Python 手搓一个轻量级的平替版 MCP Server：MiniSynthPilot！

核心实现：TCP 通信与 Tcl 交互

我们知道，Vivado 提供了非常强大的 Tcl 交互接口。如果使用 batch 模式，每次调用都会启动和关闭 Vivado，不仅慢，而且无法直观地看到工程界面的变化。

因此，我们的思路是：

  1. 启动常驻的 Vivado GUI：用 Python subprocess 拉起一个 Vivado 图形界面，并让它运行一个极其轻量的 Tcl TCP Server。
  2. 监听 9999 端口：这个 Tcl Server 在后台默默监听，接收来自 Python 的指令。
  3. MCP 工具封装：使用 FastMCP 库，把“创建工程”、“添加文件”、“综合”、“仿真”等常用操作封装成工具，通过 Socket 将具体的 Tcl 脚本发送给 Vivado 执行。

这样，当我向大模型下达指令时，我的屏幕上就会直接弹出 Vivado 界面，并自动执行诸如建工程、加代码、跑波形等操作，一切尽在掌握！

踩坑记录：Windows 下的编码乱码

在开发过程中，我们遇到了一个非常典型的坑。

当 AI 调用 MCP 生成带中文输出（比如 puts "✅ 实现完成！"）的 Tcl 脚本并通过 Socket 发送给 Vivado 时，Vivado 的 Tcl Console 里出现了一堆“火星文”乱码。
